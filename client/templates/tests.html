This is for testing. You can force the client direct to request new surveys from the server (if available), to match incoming server inquiries with local client inquiries, and to send reports (if any matches apply) back to the server. This is usually done automatically and can be configured in internal/config.py

Forces the client to request surveys from the server. If surveys are availalbe, they will be fetched by ther server and each question will be stored indivdually in 'server inquiries'. If 'quizmode' ist activated, questions from the survey will be stored in client inquiries as well.


A match between client- and server inquiries will be made, if the the corresponding client inquiries are not locked by the user. If name, type and options are identical between client- and server inquiries a report will be generated under the relating surveyid.
One report will be created for each call of 'match inquiries'.


Forces the client to send a report (if available) to the server. If a report was sent, it will be deleted from the client. Server inquiries belongig to the surveyid of the report will be deleted as well.
One report will be sent for every call of 'send report'.


Deletes all archive entries.
If a survey is stored in the archive, no new surveys with the corresponding surveyid will be accepted.
